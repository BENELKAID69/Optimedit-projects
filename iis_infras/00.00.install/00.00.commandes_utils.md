
# Correction de fichiers
#------------------------
# lorsque le fichier possède des retours à la ligne de type Windows (CRLF) au lieu de LF (Linux), 
# ce qui empêche le shell de trouver l'interpréteur /bin/bash correctement.

Lors de la création de scripts Bash sur une machine Windows, veillez à toujours configurer l'éditeur de 
texte en "End of Line: LF" (et non CRLF). Si vous rencontrez l'erreur 
"le fichier requis n'a pas été trouvé" alors que le fichier existe, 
utilisez la commande sed -i 's/\r$//' 
# ex: 
sudo sed -i 's/\r$//' install.sh
