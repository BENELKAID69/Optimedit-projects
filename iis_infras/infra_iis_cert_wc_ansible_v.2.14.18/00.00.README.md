Ce dépôt Git centralise l'ensemble de l'Infrastructure as Code (IaC) et des scripts de recette permettant de:
déployer, sécuriser, certifier et durcir une infrastructure web composée de **13 applications métiers** sur le domaine `optimedit.eu`.

Les technoligies et protocoles utilisés: AD, ADCS, DNS, SMB, IIS, Kerberos (Windows et Linux), Délégation Kerberos, Ansible Playbooks

# ===========================================================================================================================
📂 Partie 1 : Installation et Prérequis Système (00.00.install)
# ===========================================================================================================================

Ce dossier contient la fondation "Out-of-Band" de l'infrastructure, nécessaire avant toute exécution des playbooks Ansible.
1.1. 00.01.windows/ : Fondations de l'Annuaire & PKI

    AD/ : Contient la logique d'administration Active Directory.

        01. (GPO) Configuration services drives/ : Automatisation du provisionnement des serveurs de fichiers. Ce dossier centralise les scripts de déploiement d'infrastructure de fichiers (création de groupes AD imbriqués : locaux, globaux, universels) et la gestion des partages sécurisés par GPO.

    ADCS/ : Cœur de la chaîne de confiance (PKI).

        01. (ADCS) PKI/ : Documentation et fichiers de configuration pour l'installation et le durcissement de l'Autorité de Certification.

        02. Gen-Cert/ : Moteur de création des modèles de certificats. Ce répertoire est crucial : il gère la génération des templates pour les services IIS et WinRM. Il propose des méthodes granulaires pour le déploiement de certificats Wildcard (PFX) ou FQDN (avec/sans auto-enrôlement), garantissant la sécurité des communications HTTPS et WinRM.

        03. Ex-Import-CA3-To-Ansible/ : Procédures d'exportation de la racine de confiance (Root CA) vers le Master Ansible pour permettre une communication sécurisée.

1.2. 00.02.ansible/ : Initialisation du Master Ansible

Ce dossier contient les outils d'amorçage (bootstrapping) de ton serveur d'automatisation.

    00.00.setup_complet_multi_admin_ansible_V7_APT-No-PIP.sh : Installation standard d'Ansible via le gestionnaire de paquets système (apt). Recommandé pour une stabilité maximale.

    00.00.setup_complet_multi_admin_ansible_V7_PIP.sh : Installation flexible d'Ansible via pip (Python), permettant de gérer précisément les dépendances et les versions des bibliothèques nécessaires au projet.

    00.00.add_new_ansible_admin_V3.sh : Script utilitaire pour la gestion des accès administrateurs sur le Master, assurant la traçabilité des interventions.

# ===========================================================================================================================
📂 Partie 2 : Orchestration et Déploiement (infra_iis_cert_wc_ansible_v.2.14.18)
# ===========================================================================================================================

Ce dossier est le "cerveau" qui exécute l'automatisation une fois les prérequis installés.

    00.00.audit_winrm_certs/ : Outils de diagnostic pour vérifier l'état des certificats WinRM sur l'ensemble de ton parc.

    01.00.plays_ad_dns_delegation_spn/ : Phase critique de préparation de l'AD, incluant la création des comptes de service, la gestion des zones DNS et surtout la délégation Kerberos (SPN) pour le fonctionnement du "Double Hop".

    02.00.plays_install_iis_pool_http/ : Installation automatisée des rôles IIS et déploiement du contenu applicatif de base.

    03.00.plays_conf_https_443_ports_dedies/ : Configuration du durcissement HTTPS, incluant le nettoyage (flush) des anciennes liaisons et l'injection des certificats.

    04.00.plays_pool_indentity_svc_acl_ntfs/ : Sécurisation avancée des applications via l'utilisation d'identités de services dédiées et le verrouillage des droits NTFS.

    05.00.plays_pools_recycling_maintenances/ : Gestion du cycle de vie des processus IIS (logs, recyclage nocturne, maintenance ASP.NET).

    roles/ & group_vars/ : Organisation métier des composants (DC, FS, IIS, ADCS) et centralisation des variables/secrets pour une cohérence totale de l'infrastructure.
	

Plus d'info:
1. Configuration & Structure (Répertoire racine)

    group_vars/ : Configuration globale et secrets chiffrés.

        all.yml : Variables communes à l'infrastructure.

        srv_*.yml | srv_*_vault.yml : Configuration et secrets (Ansible Vault) par rôle (ADCS, DC, FS, IIS).

    host_vars/ : Paramètres spécifiques par serveur (ex: OPT-IIS-XX.optimedit.eu.yml).

    roles/ : Logique métier des composants.

        adcs_setup/ : Gestion de l'Autorité de Certification.

        dc_setup/ : Configuration AD, DNS et SPNs (via 02.03.Update-IIS-SPNs.ps1).

        fs_setup/ : Sécurisation des partages et accès fichiers.

        iis_setup/ : Configuration IIS et templates PowerShell (configure_pool.ps1.j2, copy_cert.ps1.j2).

2. Séquence de Déploiement (Playbooks)
01.00.plays_ad_dns_delegation_spn/

    01.01.play_iis_dc_create_users_applications.yml

        > Info : Initialise l'identité applicative dans l'AD. Indispensable pour la séparation des privilèges.

    01.02.play_iis_dc_setup_dns.yml

        > Info : Configure les enregistrements DNS requis pour la résolution des noms des sites hébergés.

    01.03.play_iis_dc_ad_group.yml

        > Info : Crée les groupes de sécurité AD pour la gestion granulaire des droits NTFS.

    01.04.play_iis_dc_ad_spn.yml

        > Info : Point d'entrée pour le "Double Hop". Les SPN permettent la délégation Kerberos vers le FS.

    01.05.play_iis_test_kerberos_delegation_smb_write.yml

        > Info : Teste la chaîne de délégation. Si ce test échoue, le déploiement des certificats via FS sera bloqué.

    01.07.play_iis_maintenance_audit_applications_SPN_HTTP.yml

        > Info : Audit final de la configuration Kerberos pour s'assurer de l'absence d'erreurs de duplication SPN.

02.00.plays_install_iis_pool_http/

    02.01.play_iis_install_http.yml

        > Info : Installation du rôle IIS. Nécessite que le serveur soit membre du domaine.

    02.02.play_iis_generate_index.html.yml

        > Info : Génération dynamique des pages de test index.html pour valider la connectivité Web.

03.00.plays_conf_https_443_ports_dedies/

    03.01.play_iis_flush_ssl_before_config.yml

        > Info : Purge les liaisons SSL existantes (port 443) pour éviter les conflits avant reconfiguration.

    03.02.play_iis_ssl_https.yml

        > Info : Liaison du certificat Wildcard. Si délégation impossible, le cert doit être sur le Master Ansible.

    03.03.play_iis_hardening_port_dedies_ssl_flush.yml

        > Info : Nettoyage des liaisons SSL sur les ports dédiés pour isoler les services HTTPS.

    03.04.play_iis_hardening_port_dedies_ssl_conf.yml

        > Info : Applique le durcissement SSL sur les ports spécifiques (Protocoles/Ciphers).

04.00.plays_pool_indentity_svc_acl_ntfs/

    04.01.play_iis_hardening_pool_svc.yml

        > Info : Bascule l'exécution des Application Pools sur les comptes de service AD (sécurité accrue).

    04.02.play_iis_acl_applications.yml

        > Info : Applique des ACL NTFS restrictives basées sur les groupes AD créés à l'étape 01.03.

    04.03.iis.test_dir_acl_apps_Use_--limit_cible.yml

        > Info : Vérification de conformité des permissions (Audit).

05.00.plays_pools_recycling_maintenances/

    05.01.play_iis_maintenance_AHH_AHS_Local_Keys_aspnet_regiis.yml

        > Info : Enregistrement des outils ASP.NET pour assurer le support des clés de chiffrement locales.

    05.02.play_iis_maintenance_enable_recycling_logs.yml

        > Info : Active la journalisation détaillée des recyclages IIS pour le diagnostic de performance.

    05.03.play_iis_maintenance_configure_nightly_recycle.yml

        > Info : Planifie le recyclage automatique nocturne pour la stabilité mémoire.

    05.04.play_iis_audit_pools_recycling_status.yml

        > Info : Vérification de l'état de configuration du recyclage sur les pools.

    05.05.play_iis_audit_was_activity_logs.yml

        > Info : Audit de l'activité du service WAS via l'Event Viewer (logs de bas niveau).
